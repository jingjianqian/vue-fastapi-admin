# 前端调试指南

## 问题诊断步骤

### 1. 打开浏览器开发者工具
按 F12 打开开发者工具

### 2. 检查控制台（Console）错误
查找以下类型的错误：
- 组件加载失败
- 路由匹配失败  
- vueModules undefined

### 3. 检查网络请求（Network）
- 点击小程序管理菜单后
- 查看是否有 /api/v1/wechat/list 请求
- 如果没有请求，说明组件未渲染

### 4. 在控制台执行以下命令检查路由

```javascript
// 检查当前路由
console.log("当前路由:", $route)

// 检查所有注册的路由
console.log("所有路由:", $router.getRoutes())

// 查找 wechat 相关路由
$router.getRoutes().filter(r => r.path.includes('wechat'))
```

### 5. 检查权限 store
```javascript
// 检查生成的访问路由
console.log("访问路由:", usePermissionStore().accessRoutes)

// 检查菜单
console.log("菜单:", usePermissionStore().menus)
```

## 快速修复方案

### 方案 1: 重启 Vite 开发服务器（推荐）

Vite 的 glob 导入在启动时解析，新添加的文件可能未被识别。

1. 停止前端服务 (Ctrl+C)
2. 清除缓存并重启:

```pwsh
cd web
rm -rf node_modules/.vite
npm run dev
```

### 方案 2: 强制刷新浏览器
Ctrl+Shift+R 或 Ctrl+F5 强制刷新，清除缓存

### 方案 3: 检查文件是否被 git 忽略
```pwsh
git status web/src/views/system/wechat/
```

如果文件被忽略，Vite 可能不会监听它
